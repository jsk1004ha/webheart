Implement a reference-corrected rPPG webcam measurement workflow for this repository.

Target result:
- This is not simple webcam heart-rate measurement. Implement a measurement-method improvement study: simultaneously measure skin ROI and reference ROI in a single webcam video, use skin/reference reflected-light ratio to reduce common illumination and camera auto-exposure noise, then compare it to baseline rPPG methods.

Required research/report framing in Korean:
- Recommended title: "피부 ROI와 기준 ROI의 반사광 비율을 이용한 참조 보정형 rPPG 심박 신호 추출 연구"
- Core inquiry: "피부 ROI와 기준 ROI의 반사광 비율을 이용하면, 단일 Green 채널 방식보다 조명 변화와 카메라 자동 노출에 강한 rPPG 심박 신호를 추출할 수 있는가?"
- Explain reference ROI purpose, common illumination-noise cancellation principle, relative reflectance/absorbance interpretation, and comparison with 기존 Green 방식.

Functional requirements:
1. User selects skin ROI and reference ROI from the first frame.
2. skin ROI is forehead/cheek exposed skin; reference ROI is wall/clothes/background/hair/non-blood-flow region.
3. Store per-frame mean R,G,B for both ROIs.
4. Compute methods:
   - raw Green: s_raw(t) = G_skin(t)
   - normalized Green: g_skin(t), normalized from skin RGB itself
   - reference absorbance channels:
     A_R(t) = -log((R_skin(t)+eps)/(R_ref(t)+eps))
     A_G(t) = -log((G_skin(t)+eps)/(G_ref(t)+eps))
     A_B(t) = -log((B_skin(t)+eps)/(B_ref(t)+eps))
   - reference-corrected Green: s_ref_green(t) = A_G(t)
   - reference-corrected RGB difference: s_ref_rgbdiff(t) = 2*A_G(t)-A_R(t)-A_B(t)
5. For each method, apply band-pass filter, FFT, detect f_peak, BPM = 60*f_peak, peak_strength, and SNR-like/peak-quality metric.
6. Produce side-by-side comparison graphs: time-domain signals and FFT spectra for baseline and reference-corrected methods.
7. Produce report_summary.txt in Korean covering: why reference ROI is used; how common lighting variation is removed; results compared to existing method; suggested experiments (lighting-change robustness and reference ROI location comparison if supported).
8. Keep implementation aligned with the repo’s existing patterns and UI/CLI structure. Avoid unnecessary new dependencies.
9. Verify with available tests/lint/build or the smallest meaningful local validation.

Completion criteria:
- rPPG processing supports both skin and reference ROI.
- All required methods and metrics are implemented.
- Output graph/report artifacts are generated by the app/script.
- Verification evidence is recorded.
